# CLAUDE.md — Инструкции для Claude Code

## Контекст проекта

**PsyFusions** — лендинг психолога и гипнолога Оксаны Миру.

### Две версии проекта
| | Статичная (текущая) | React-версия |
|---|---|---|
| Файлы | `index.html` + `styles.css` + `script.js` | `src/App.tsx` + `src/components/Quiz.tsx` |
| Запуск | Открыть `index.html` в браузере | `npm run dev` |
| Стек | Tailwind CDN + Vanilla JS | React 19 + Vite + Framer Motion |

**Основная рабочая версия — статичная** (`index.html`).

Сайт должен вызывать **доверие, тепло и профессионализм**.
Главная цель — привести посетителя к заявке или контакту в мессенджер.

## Ключевые принципы работы

### 1. Не ломать структуру
- Не перемещай и не переименовывай файлы без явной просьбы
- Не удаляй секции лендинга без подтверждения
- Не меняй маршруты навигации (`#methodology`, `#about`, и т.д.)
- Не меняй структуру `src/config.ts` — только значения внутри

### 2. Сохранять визуальную концепцию
- **Цветовая палитра**: тёмно-зелёный (`#3D4435`), терракота (`#BC6C25`), кремовый (`#FDFCF9`) — не менять без явного запроса
- **Типографика**: serif для заголовков, sans для текста — сохранять соотношение
- **Rounded corners**: карточки — 40px, кнопки — 16-24px, hero-фото — 100px — сохранять стиль
- **Анимации**: Framer Motion (`motion`, `AnimatePresence`) — не заменять на CSS-только

### 3. Вносить изменения аккуратно
- Читай файл перед редактированием
- Меняй только то, о чём просят — не рефакторь окружающий код
- Сохраняй существующие комментарии в коде
- Не добавляй лишние зависимости без необходимости

### 4. Делать код чистым и понятным
- TypeScript типы для новых компонентов
- Понятные имена переменных на английском
- Комментарии на русском — для бизнес-логики
- Не дублируй — если функция уже есть, используй её

### 5. Ориентироваться на конверсию
Любое изменение должно работать на одну из целей:
- **Доверие**: выглядеть профессионально и тепло
- **Понимание**: объяснять, что и как делает психолог
- **Действие**: вести к заявке, кнопке «Записаться», мессенджеру

## Компоненты и их роль

### `Navbar`
Фиксированная навигация. Меняет стиль при скролле. Мобильное меню через AnimatePresence.
**Не трогай** z-index (z-50) и backdrop-blur — они важны для перекрытия контента.

### `SectionHeading`
Переиспользуемый заголовок секции: subtitle (маленький, uppercase) + title (большой, serif).
`centered` prop для центрирования.

### `Card`
Белая карточка с rounded-[40px]. Используется во всех секциях.

### `Quiz`
Отдельный компонент в `src/components/Quiz.tsx`.
Пять шагов: intro → questions → calculating → result → form.
Отправляет данные на `/api/contact`.

## Частые задачи

В `src/App.tsx`, секция `testimonials`, в массиве:

В `src/App.tsx`, секция `requests`, в массиве:

```tsx
{ title: 'Название', desc: 'Описание.', icon: <IconComponent /> }
```

## Важные ограничения

- **Не используй** `@google/genai` без явного запроса — эта зависимость оставлена от AI Studio
- **Не меняй** `server.ts` без понимания — там обрабатываются заявки клиентов
- **Не трогай** `AI-google-studio/` папку — это оригинальный архив
- **Не добавляй** новые npm пакеты без проверки, нет ли уже аналога в проекте
- **Tailwind v4**: синтаксис отличается от v3 — `@theme {}` вместо `theme.extend` в конфиге
